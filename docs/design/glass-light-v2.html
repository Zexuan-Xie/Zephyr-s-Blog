写一个能读懂的全栈 blog

2026-05-30 · 8 min read · Go / React / Postgres

前后端解耦不是为了赶时髦，而是想亲眼看到那条 HTTP 边界——SSR 框架刻意把它抹掉了。

第一次做全栈，我的目标不是"从零默写"，而是"能读懂、能判断、能改" AI 生成的每一块。Go + chi，因为 handler 是原生 net/http 签名，学到的是所有 Go web 框架的公分母，每一行都透明可读。

func main() {
  r := chi.NewRouter()
  r.Use(middleware.Logger)
  r.Get("/api/tree/resolve", resolvePath) // 公开内容树
  http.ListenAndServe(":8080", r)
}
